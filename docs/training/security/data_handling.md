# Data Handling & Classification

## Classification Levels

### Level 1: Public
Information intended for public release.
* *Examples*: Marketing materials, published press releases.

### Level 2: Internal
Information for internal use only.
* *Examples*: Employee directory, internal wikis, non-sensitive policies.

### Level 3: Confidential
Sensitive information that requires restricted access.
* *Examples*: Source code, architectural designs, financial reports.

### Level 4: Restricted (PII/SPII)
Highly sensitive information including Personally Identifiable Information.
* *Examples*: User passwords, SSNs, credit card numbers, health data.

## Handling Requirements
* **Encryption**: Level 3 and 4 data must be encrypted at rest and in transit.
* **Access Control**: Need-to-know basis.
* **Retention**: Delete data when no longer needed per the [Document Retention Policy](../../policies/document_retention_policy.md).
