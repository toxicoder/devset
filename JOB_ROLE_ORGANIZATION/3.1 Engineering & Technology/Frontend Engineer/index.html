    Frontend Engineer

    Role Code: SWEN1003

    Job Description

    A creative and technical role dedicated to building intuitive and responsive user interfaces for web and mobile applications. The Frontend Engineer translates design mockups into high-quality code, ensuring a seamless user experience across different devices and browsers. They optimize application performance, manage application state, and ensure accessibility standards are met. This role requires close collaboration with designers and backend engineers to bring product visions to life.

    Responsibilities

    • UI/UX Implementation: Translate design mocks into responsive, accessible (WCAG), and interactive web interfaces using React/Vue/Angular.
    • State Management: Architect efficient global state management solutions (Redux, Context API, Zustand).
    • Performance Optimization: Minimize bundle sizes, optimize rendering cycles, and improve Core Web Vitals.
    • Testing: Write unit, integration, and end-to-end tests to ensure UI stability.
    • Design System: Contribute to and maintain a shared component library for UI consistency.

    Role Variations

    • Design Technologist: Focuses heavily on the intersection of design and code, prototyping, and animations.
    • Infrastructure Frontend Engineer: Focuses on build tools (Webpack, Vite), CI/CD for frontend, and developer tooling.
    • Mobile Web Specialist: Focuses on PWA (Progressive Web Apps) and optimizing mobile web experiences.

    Average Daily Tasks

    • 10:00 Standup
    • 11:00 React coding
    • 14:00 Pixel review

    Common Partners

    Designer, Backend Eng

They treat "jank" as a critical bug. * The Accessibility Advocate: Ensures the application is usable by everyone, regardless of ability. They insist on semantic HTML, proper ARIA attributes, and keyboard navigability. They regularly test with screen readers and audit color contrast ratios to ensure WCAG compliance. * The State Architect: Deeply concerned with how data flows through the application. They carefully design state management strategies to avoid prop drilling and unnecessary re-renders. They prefer deterministic state machines and clear separation of concerns between UI and logic. * The Toolsmith: Focuses on the developer experience and build pipeline. They optimize Webpack/Vite configurations, linting rules, and CI/CD workflows to make the team more productive.     Example Phrases

    • "The user feedback on this interaction needs to be snappier; let's add an optimistic UI update."
    • "I'm worried about the cumulative layout shift on mobile; we need to reserve space for these images."
    • "Does this component support keyboard navigation? I can't tab to the close button."
    • "We should use a virtual list here because rendering 10,000 DOM nodes is freezing the main thread."
    • "The contrast ratio on this button is only 3:1; we need to darken the background to meet AA standards."
    • "Let's lazy load this heavy chart component so it doesn't block the initial page paint."
    • "I noticed a re-render loop in this component; we need to memoize these callback functions."
    • "This logic belongs in a custom hook, not inside the view component."
    • "We need to ensure all our images have alt text for screen reader users."
    • "Let's upgrade to the latest version of React to take advantage of the new concurrent features."
    • "The padding here is 15px, but the design system says it should be 16px (1rem)."
    • "We should implement a skeleton loader state to improve the perceived performance."
    • "I'm setting up a Cypress test to verify this user flow across different viewports."
    • "Please use semantic <button> tags instead of <div> with onClick handlers."
    • "We can reduce our bundle size by tree-shaking this library and only importing what we need."
